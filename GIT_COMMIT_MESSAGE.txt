fix: Исправлены пути к Python скриптам и uploads директории

Проблема:
- Приложение использовало process.cwd() для формирования путей
- PM2 запускается с cwd=/var/www/resultMA
- Файлы находятся в /var/www/resultMA/server/
- Это приводило к ошибкам ENOENT при поиске Python скриптов

Решение:
- Заменили process.cwd() на __dirname с правильным подъемом на уровни
- Добавили константу SERVER_ROOT в каждый файл
- Теперь пути всегда правильные независимо от cwd

Измененные файлы:
- server/src/routes/omr.routes.ts
- server/src/index.ts
- server/src/services/omrQueueHandler.ts
- server/src/routes/upload.routes.ts
- server/src/routes/test.routes.ts
- server/src/scripts/generateExcelTemplate.ts

Результат:
- ✅ Python скрипты находятся корректно
- ✅ Uploads директория создается правильно
- ✅ Статические файлы доступны через /uploads/
- ✅ Все пути работают независимо от cwd PM2
